---
title: "Find wallets"
description: "This endpoint is utilized to find wallet using fields(mobile, email, firstName, lastName) or a combination of them"
api: "GET /api/provider/wallets/find"
---

## Query parameters

<ParamField body="email" type="number" placeholder="chucks@parkway.ng" optional>
For this example, it is email but  the  body parameter can be one or more of this keynames below and their respective values

  - email (valid email address),
  - firstName,
  - LastName,
  - mobile (11 digtis)

  Combine the restriction to get a proper filter for finding wallets
  e.g
  To find with email and phone constraint then body should be
  ```json
  {
    "email": "chucks@parkway.ng",
    "phone": "08017123843"
  }
  ```
</ParamField>

<Warning>Passing an empty json body will lead to spooling of all wallet, for optimal performance, please make use of the body parameters to find wallets</Warning>

<ParamField query="pageSize" type="number" default="20" optional>
  it's Advisable to keep it at a max 20
</ParamField>

<ParamField query="page" type="number" placeholder="1" default="1" optional>
  You can skip by page, Default page is 1 (first page)
</ParamField>
